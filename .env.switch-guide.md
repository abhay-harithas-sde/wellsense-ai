# Environment Switching Guide

This guide helps you quickly switch between development and production configurations in your unified `.env` file.

When deploying to production, update these values in `.env`:

```bash
MONGODB_URI="mongodb://admin:ObuO59m-9T}2w7|A|t%},e)B?njYHNi,@localhost:27017/wellsense_ai?authSource=admin"

# 3. JWT Secret
JWT_SECRET=f342d55819437ef74e1cdc6c9af8eefb08df023183aa89c75cf85f235c14cd5a

# 4. Google OAuth Secret
GOOGLE_CLIENT_SECRET=V47sJQjJni/qYdAID1whJ+Rx3tK5LmqGa2gZ0eiPPM5ZVkcl

# 5. Callback URL
GOOGLE_CALLBACK_URL=https://wellsense-ai.com/auth/google/callback

# 6. CORS Origin
CORS_ORIGIN=https://wellsense-ai.com,https://www.wellsense-ai.com

# 7. HTTPS
ENABLE_HTTPS=true

# 8. Feature Flags
VITE_PRODUCTION_MODE=true
```

## Security Reminders

- Never commit `.env` to version control
- Set file permissions: `chmod 600 .env` (Linux/Mac) or restrict access on Windows
- Back up your `.env` file securely
- Rotate production secrets every 90-180 days
- Use a secrets manager (AWS Secrets Manager, Vault) for production deployments
